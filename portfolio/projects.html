This presentation was created for a group project in my Advanced Data Structures (CSC-2710) course. It showcases my understanding of the selection sort algorithm, a simple sorting algorithm that takes an array, S[n], and the size of the array as an integer, n, as inputs. Using these parameters, it keeps track of the index of the smallest element of the array and recursively swaps it with the element at S[i]. Included in the presentation is a demonstration of the algorithm, along with the C++ code that I used to implement it.

This slideshow was also created for a project in my Advanced Data Structures (CSC-2710) course. It presents the idea behind password hashing, but with a bit of a twist. Not only did I explain password hashing, my implementation of it, and the code that I used, but I encouraged users to choose between two modes: "Quick Mode" and "Funny Mode." The former, selected with (q), asks users to make a password that is hashed and compared with its re-entry's hash to verify it; the latter, selected with (f), gives users the same information, but only after asking if they want to see a user-provided character in three different number systems.

This document was created for a group project in my Computer Systems (CSC-2410) course. In short, it is a summary of Section 5.5 in the Dive Into Systems textbook by Suzanne J. Matthews, Tia Newhall, and Kevin C. Webb. It covers how a basic CPU works, including the ALU, short for Arithmetic Logic Unit, the Register File, and how the two components interact with each other. I made this summary as a collection of notes for me and my group members to reference in preparation for the project itself; this was made solely for research purposes and was not required by the professor.

This document, as stated above, was made for a group project in my Computer Systems (CSC-2410) course. It is a reflection on the project itself, which required students to design a simple Arithmetic Logic Unit and Register File from scratch in a program called Logisim — a program made for simulating circuitry in a straightforward manner. This requires a solid understanding of not just how a CPU works as a whole, but the understanding of the complex parts that make one up.
